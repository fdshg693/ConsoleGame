# ドキュメント作成の基本方針
- ドキュメントは必ず簡潔に箇条書きで書いてください。
- ドキュメントが肥大化してきた場合は、記載する内容の粒度を見直して、必要に応じてサブドキュメントを作成してください。
  - 最大行数は200行程度を目安にしてください。
- 古い内容は残さないで、常に最新の内容に更新してください。（仕様変更の理由などは書かず、最新の仕様のみを書く）

## 粒度の見直し例 
あらゆるファイルについてコメントを記載していた。
→ 重要なファイルや複雑なファイルに絞ってコメントを記載するようにした。

## サブドキュメントの作成例 
- 同階層の複数のファイルに責務に応じてドキュメントを作成する。
- `CLAUDE.md` などはサブフォルダの `CLAUDE.md` に内容を移せるか検討する。
  - または、`ai`フォルダ配下にドキュメントを作成して、パスインポート機能を利用して、`CLAUDE.md` からサブドキュメントを参照するようにする。
  - パスインポート機能
    - 例: `ai/CLAUDE.md` にドキュメントを作成して、`src/CLAUDE.md` から参照したい
    - `src/CLAUDE.md` に以下のように記載する
    ```
    @./../ai/CLAUDE.md
    ```
